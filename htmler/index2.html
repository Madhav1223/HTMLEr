Object Oriented Programming

Introduction to OOP

Object-oriented programming (OOP) is a computer programming model that organizes software design around data, or objects, rather than functions and logic.
OOP focuses on the objects that developers want to manipulate rather than the logic required to manipulate them. This includes manufacturing simulation and mobile apps.
OOP benefits collaborative development, and promotes code reusability, scalability and efficiency.

Objects belong to classes that define their data and behavior. They interact through well-defined interfaces called messages.

Features of OOP

1. Emphasis is on data rather than procedures.
2. Programs are divided into objects.
3. Function and data are tied together in a single unit.
4. Data access is through visible functions enabling object communication.
5. Data structures are modeled as objects.

Class

A Class is a user-defined data-type containing data members and member functions defining properties and behavior of its objects.

Object

An Object is an instance of a Class that occupies memory and has attributes and behaviors defined by its class.

Encapsulation

Encapsulation binds data and functions into a single unit, hiding internal details and only exposing necessary components.

Inheritance

Inheritance allows a new class to acquire the properties and behaviors of an existing class.

Polymorphism

Polymorphism enables objects to be treated as instances of their parent class, supporting multiple forms of behavior.

Abstraction

Abstraction reduces complexity by hiding unnecessary details and exposing only essential features.
